# 9. SSG

## Que aplica realmente al proyecto

FitTrack se centra actualmente en gestion de rutinas y flujo de entrenamiento personal.  
La parte SSG se documenta con honestidad para no atribuir funcionalidades no implementadas.

## Gestion de usuarios/clientes

- Existe flujo de acceso en frontend con token simulado.
- No hay gestion completa de usuarios/clientes con autenticacion backend real en esta fase.

## Formularios

- Formularios implementados para crear/editar rutinas.
- Formulario implementado para registrar entreno desde rutina.

## Paneles de control

- No existe panel administrativo completo en esta fase.

## Estado real

- **Implementado:** formularios funcionales del dominio principal (rutinas y registro de entreno).
- **Parcial:** autenticacion temporal sin gestion completa de usuarios.
- **Futuro:** paneles de control y gestion avanzada de usuarios/clientes.
